# 제Ⅰ장 서론 (v5 재작성)

## 1절 연구 배경

검색 증강 생성(Retrieval-Augmented Generation, 이하 RAG)은 대형 언어 모델(LLM)의 환각과 지식 단절 문제를 외부 지식 주입으로 완화하는 핵심 기법으로, 2020년 이래 산업·학계 양면에서 급속히 확산하였다. 단일 소스 RAG의 한계가 실증되면서 복수의 지식 표현(텍스트 벡터, 지식 그래프, 형식 온톨로지)을 결합하는 하이브리드 RAG가 대두되었으며, 본 연구의 저자는 선행 논문(Shin & Moon, 2025, JKSCI)에서 **Triple-Hybrid RAG** 프레임워크와 **Rule-based Dynamic Weighting Algorithm (R-DWA)** 을 제안한 바 있다.

선행 연구의 R-DWA는 질의 유형 분류와 개체/관계/제약 밀도 신호를 2단계로 결합하여 세 소스의 가중치 (α, β, γ)를 동적으로 결정한다. 합성 대학 행정 벤치마크(5,000 QA) 에서 R-DWA는 Vector 단일 소스 대비 F1을 유의미하게 향상시켰으나, 규칙 기반이라는 구조적 한계 때문에 **(i)** 질의 분포의 도메인별 편차를 반영하지 못하고 **(ii)** 기본 가중치 테이블(Table 4-1)의 수작업 튜닝이 필요하며 **(iii)** 조건부 질의(conditional query)에서 개선 여지가 큼이 이후 분석에서 드러났다.

본 박사 논문은 이 구조적 한계를 **학습형 동적 가중치 알고리즘(Learned DWA, 이하 L-DWA)** 로 해소한다. 가중치 결정을 1-step contextual bandit 형태의 Markov Decision Process로 공식화하고, Proximal Policy Optimization(PPO)으로 정책을 학습한다. 실험의 비용 장벽은 **오프라인 보상 캐시**로 해결하여, 학습 중 LLM API 호출이 0이 되는 구조를 제시한다.

> **[박스 1-1] Triple-Hybrid RAG — 한눈에 보기**
>
> **정의.** 하나의 자연어 질의에 대해 **세 가지 지식 소스** 를 병렬 조회하고 각 소스의 기여도를 가중치로 융합하여 최종 컨텍스트를 구성하는 RAG 아키텍처.
>
> **세 소스.**
>
> | 소스 | 표현 | 가중치 | 강점 |
> |---|---|---|---|
> | Vector | 문서 임베딩 (FAISS) | α | 의미 유사도 |
> | Graph | 엔티티-관계 그래프 (NetworkX, BFS) | β | 다단계 관계 |
> | Ontology | OWL 클래스·제약 (Owlready2) | γ | 논리 제약 |
> | 제약 | α + β + γ = 1, 모두 ≥ 0 (단순체 Δ³) | | |
>
> **예시.** "기계공학과 55세 이하 교수는?" 질의에 대해:
> - (α, β, γ) = (0.33, 0.33, 0.33) 균등 → 평균 성능
> - (α, β, γ) = (0.1, 0.2, 0.7) Ontology-heavy → 55세 제약 적용 ↑
>
> **본 논문의 핵심 질문.** 이 (α, β, γ) 를 **질의마다 어떻게 최적 결정하는가?** — 선행의 R-DWA(규칙) 대비 학습된 L-DWA(PPO) 로 개선 가능한가?

## 2절 연구 문제

본 논문은 다음 세 가지 연구 질문(RQ)에 답한다.

**RQ1.** 규칙 기반 R-DWA의 고정된 기본 가중치 테이블이 실제 벤치마크의 보상 지형(reward landscape)과 정렬되어 있는가? 정렬되지 않았다면 그 간극은 정량적으로 얼마인가?

**RQ2.** Triple-Hybrid 가중치 결정을 MDP로 공식화하고 PPO로 학습한 L-DWA는 R-DWA 대비 어떤 정량적 개선을 제공하는가? 개선은 질의 유형별로 어떻게 분포하는가?

**RQ3.** 특정 도메인(합성 대학)에서 학습된 L-DWA의 정책은 타 도메인(HotpotQA, MuSiQue, PubMedQA)으로 이전 가능한가? 이전 가능성/불가능성은 본 프레임워크의 가치 제안에 어떤 시사점을 주는가?

## 3절 연구 기여 — 4단계 구조

본 논문의 기여는 **선행 JKSCI 2025 논문의 재현성 장애 3가지를 차례로 해소하여 신뢰 가능한 새 기준점을 수립하고, 그 위에 PPO 기반 L-DWA 를 도입하는** 4단계로 구성된다. 단계별 수치는 `docs/CORRECTED_BASELINE.md` 에 단일 출처로 기록된다.

### 3.1 재현성 기여 (Reproducibility) — Stage 1

1. **선행 논문 평가기 버그의 자발적 식별 및 정정.** 선행 논문의 `evaluator.py::normalize()` 가 구두점을 토큰화 전에 제거하지 않아, 콤마 구분 리스트 gold (`"홍성민, 황성민, 전성민"`) 의 토큰이 `{"홍성민,", "황성민,", "전성민"}` 으로 분리되어 prediction 토큰 `{"홍성민", ...}` 과 교집합이 **마지막 토큰 하나로 축소** 되는 버그를 발견하여 `_PUNCT_RE` 패치로 수정. 본 저자가 2026-04-22 선행 저장소에 [**CORRIGENDUM**](https://github.com/sdw1621/hybrid-rag-comparsion/blob/main/CORRIGENDUM.md) 을 공개 정정 (Ch.6 §1.4).
2. **Table 8 노트북 셀 미실행 증거 보고.** 선행 저장소의 `notebooks/Triple_Hybrid_RAG_Full.ipynb` Step 8 의 `execution_count: null, outputs: []` 상태를 CORRIGENDUM 에 기록하여 원 논문의 5,000 QA 전수 실험 증거 부재를 투명히 공개.

### 3.2 측정 방법론 기여 (Methodological Evaluation) — Stage 2, 3

3. **프롬프트-gold 형식 정렬.** `PROMPT_TEMPLATE_LIST` 를 도입하여 LLM 출력을 gold 의 콤마 구분 리스트 형식과 정합 (S1 위에서 R-DWA F1<sub>strict</sub> 0.137 → **0.529 (+286%)**, EM 0 → **0.387** 구조적 해소, Ch.6 §3.4).
4. **2-branch Faithfulness.** 리스트형 응답에 대해 각 콤마 항목을 **개별 claim** 으로 검증하는 분기를 추가하여 환각 탐지 해상도를 상승. Sentence 분기는 하위 호환 유지 (Ch.6 §3.5).
5. **삼중 F1 체계.** 엄격 토큰 집합 F1 (F1<sub>strict</sub>), 콤마 항목 부분문자열 F1 (F1<sub>substring</sub>), 문자 3-gram F1 (F1<sub>char</sub>) 을 병기 — 평가 민감도의 투명화 (Ch.6 §1.3).

### 3.3 학습 방법론 기여 (Methodological Learning) — Stage 4

6. **MDP 공식화.** Triple-Hybrid RAG 의 가중치 결정을 **18차원 상태 공간 × 3-simplex 행동 공간의 1-step MDP** 로 공식화한 최초의 정식화 (Ch.5 §1).
7. **PPO 기반 L-DWA.** Actor-Critic 네트워크 (shared backbone 18→64→64, Dirichlet 농도 헤드, 5,636 파라미터) + Clip 0.2 PPO 학습 레시피로 **3 seeds (42, 123, 999) 전부 동일 보상 0.215 ± 0.002** 에 수렴 (Ch.5 §3, Ch.6 §5).

### 3.4 엔지니어링 기여 (Engineering)

8. **오프라인 보상 캐시.** 5,000 질의 × 66 이산 가중치 = **330,000 엔트리** 의 reward 를 SQLite 에 사전 계산 ($33, 14시간) 하여 PPO 학습 중 API 호출 0, 3 seeds 학습을 **50시간 → 1시간 (−98%)** 으로 단축 (Ch.5 §4).

### 3.5 실증적 기여 (Empirical) — Corrected Baseline 위에서

본 논문의 핵심 비교는 **S1+S2+S3 가 모두 적용된 corrected baseline 위에서 R-DWA vs L-DWA** 의 측정이다. 모든 수치는 5,000 QA 전수 측정.

- **L-DWA 3-seed 평균 F1<sub>strict</sub> = 0.562 ± 0.007**. R-DWA corrected baseline 0.529 대비 **+6.2%** — 이것이 **순수 PPO 학습의 기여** 로, 평가기 수정(+90%) · 프롬프트 정렬(+286%) 과 구분되는 독립 이득.
- **L-DWA 가 Oracle (per-query argmax, 66-point grid) 을 모든 F1 지표에서 초과**: F1<sub>strict</sub> 0.562 > 0.554, F1<sub>substring</sub> 0.507 > 0.504, F1<sub>char</sub> 0.494 > 0.487. Dirichlet 평균이 이산 격자 외부를 탐색하여 상한을 넘어섰다.
- **Conditional 유형에서 F1<sub>strict</sub> +36.7%** (R-DWA 0.223 vs L-DWA 0.304) — per-query 학습형 가중치의 수작업 규칙 대비 **질적 우월성**.
- **3 seeds std < 1%** — 학습 알고리즘의 극단적 재현성.
- 한편 **합성 대학 도메인에서 학습된 L-DWA 를 HotpotQA, MuSiQue, PubMedQA 에 naive transfer 시 Vector-only 대비 F1 −30 ~ −35% 저하** — **도메인 특화 학습이 본 프레임워크 가치의 필수 요소** 라는 솔직한 경계 조건 (Ch.6 §7).

**정리.** 본 논문의 개선 총량은 **재현성 (S1) + 측정 방법론 (S2, S3) + 학습 알고리즘 (L-DWA)** 의 합성이며, Ch.6 에서 각 단계의 기여를 분리 보고한다. 원 JKSCI 0.86 과의 직접 비교는 CORRIGENDUM 으로 재현 불가를 공개하였으므로 본 논문은 **corrected baseline 위에서의 학습 효과** 를 핵심 주장으로 한다.

## 4절 본 논문의 범위와 한계

본 논문은 다음을 **다룬다**:
- Triple-Hybrid RAG (Vector + Graph + Ontology) 의 가중치 결정 학습
- PPO 기반 정책 학습의 수렴과 재현성
- 합성 대학 벤치마크의 5,000 QA 전수 평가
- HotpotQA/MuSiQue/PubMedQA 교차 도메인 **이전 실험** (domain transfer feasibility)
- 엄격-loose 이중 평가 기준의 비교

본 논문은 다음을 **다루지 않는다** (향후 연구 과제):
- 지식 그래프·온톨로지의 LLM 기반 자동 구축
- 다국어 intent 분석기 (현 구현은 한국어 regex 기반)
- 도메인 간 joint training 또는 meta-learning
- BERT multi-label intent classifier의 학습 (§5.2에서 구현만 제시하며 intent_logits=0 상태로 학습)
- HotpotQA에서의 L-DWA domain-specific 학습 (reward landscape cliff로 인한 학습 불가 현상을 Ch.6 §7.3에 보고)
