# CS-7. List-style 프롬프트 효과 케이스 스터디

> **소속.** Ch.Ⅵ § 3.4 *프롬프트 형식과 평가 민감도* 의 정성 보완.
> **요약.** 동일 질의·동일 가중치·동일 LLM에 대해 sentence 프롬프트와 list 프롬프트의 응답 형태·평가 지표 차이를 7개 실례로 비교한다. 형식 교정만으로 F1<sub>strict</sub> 가 약 3~10배 개선됨을 보이는 한편, Faithfulness 의 2-branch 설계(박스 6-4) 가 환각을 어떻게 노출하는지 보여준다.

---

## CS-7.1 질의 1: 단일 과목 담당 교수 (Simple)

> **질의.** "정치외교 심리학개론 과목을 담당하는 교수는?"
> **Gold.** `홍성민, 황성민, 전성민`
> **R-DWA 선택 가중치.** (α, β, γ) = (0.2, 0.6, 0.2)

### Sentence 프롬프트 응답
```
정치외교 심리학개론 과목을 담당하는 교수는 홍성민, 황성민, 전성민입니다.
```

### List 프롬프트 응답
```
홍성민, 황성민, 전성민
```

### 지표 비교

| 지표 | Sentence | List |
|---|---|---|
| **F1<sub>strict</sub>** | 0.364 | **1.000** |
| F1<sub>substring</sub> | 1.000 | 1.000 |
| **EM<sub>norm</sub>** | 0.000 | **1.000** |
| Faithfulness | 1.000 | 1.000 |
| Latency (s) | 2.08 | 0.79 |

**관찰.** 내용은 동일하나 sentence 프롬프트는 "과목을", "담당하는", "교수는", "입니다" 등의 filler 토큰 때문에 F1<sub>strict</sub> 에서 recall=1.0·precision=0.375 → F1=0.364 로 계산된다. List 프롬프트는 토큰 집합이 gold와 정확히 일치하므로 **F1<sub>strict</sub> = 1.0, EM = 1.0** 을 달성한다. 지연 역시 2.6배 감소 — 짧은 출력이 LLM generation time 을 줄인다.

## CS-7.2 질의 8: 조건부 교수 탐색 (Conditional, 부분 성공)

> **질의.** "소프트웨어공학과 소속 55세 이하 교수는?"
> **Gold.** `윤민준, 장민준, 황민준, 안민준`
> **R-DWA 선택 가중치.** (α, β, γ) = (0.2, 0.2, 0.6)

### Sentence 프롬프트 응답 (요지)
> *"소프트웨어공학과 소속 교수는 윤민준 … 과 안서준 교수 … 입니다. 그러나 안서준 교수는 전자공학과 소속 … 윤민준의 나이는 정보에 없으므로 … 따라서 소프트웨어공학과 소속 55세 이하 교수에 대한 정보는 찾을 수 없습니다. 결론적으로 … 정보가 없습니다."*

### List 프롬프트 응답
```
안서준, 윤민준
```

### 지표 비교

| 지표 | Sentence | List |
|---|---|---|
| **F1<sub>strict</sub>** | 0.059 | **0.333** |
| F1<sub>substring</sub> | 0.200 | 0.250 |
| Faithfulness (지지된 항목 / 전체) | 1.000 (문장 any-token, 인플레이션) | **0.500 (1/2, 안서준 환각 노출)** |
| Latency (s) | 3.68 | 0.74 |

**관찰.** 두 응답 모두 **정답 4명 중 1명 (윤민준)** 만 맞추고 안서준을 환각한다. Sentence 프롬프트는 5줄에 걸쳐 장황하게 해명하며 filler 토큰이 많아 문장형 faithfulness(any-token) 이 **과대평가 1.0** 을 준다. List 프롬프트는 2개 항목 중 1개만 context 에 실제 등장 → **Faithfulness 0.5** 로 환각을 정확히 반영.

**시사점.** List prompt + 2-branch Faithfulness 의 조합이 **환각 탐지 해상도** 를 높인다. 본 연구 5,000 QA 전수 측정에서 R-DWA 의 Faithfulness 가 0.835 → 0.544 로 감소한 것은 품질 저하가 아니라 평가의 **엄격화**. 이 해석은 Ch.Ⅵ § 3.5 에 명시되어 있다.

## CS-7.3 질의 5: 교수 연구 분야 목록 (Conditional, 대규모 gold)

> **질의.** "화학과 소속 40세 이상 교수의 연구 분야는?"
> **Gold.** 14개 항목 (중복 포함): `회계정보, 로봇공학, 분산시스템, 디지털미디어, 암호학, 컴퓨터비전, 머신러닝, 회계정보, 언어학, 바이오공학, 자연어처리, 컴퓨터비전, 암호학, 경영과학`
> **List 프롬프트 응답.** `머신러닝, 회계정보` (2개 항목, 둘 다 gold에 포함)

| 지표 | 값 |
|---|---|
| F1<sub>strict</sub> | 0.308 |
| F1<sub>substring</sub> | 0.214 |
| F1<sub>char</sub> | 0.207 |
| Faithfulness (리스트 분기) | **1.000** |

**관찰.** Gold list 가 대규모(14 항목) 일 때 LLM은 보수적으로 소수만 출력하는 경향. 출력한 2 항목은 모두 context 에 지지됨 → Faithfulness 만점. 그러나 recall 이 제한적이라 F1 은 중간 수준. 이는 **list prompt의 한계** — **완전한 전수 열거** 는 여전히 어려우며, future work 로 *exhaustive listing prompt* 설계 여지를 시사.

## CS-7.4 질의 3: 대용량 리스트 + gold 형식 이상 (Conditional)

> **질의.** "전임 교수 중 55세 미만인 사람은?"
> **Gold (원본).** `총 367명 (김민준, 이민준, 박민준, 정민준, 최민준…)` — **367명을 "5명 + …" 형태로 압축** 한 non-정규 표현.
> **List 프롬프트 응답.** `김민준, 이민준, 박민준`

| 지표 | 값 |
|---|---|
| F1<sub>strict</sub> | 0.600 |
| F1<sub>substring</sub> | 0.400 |
| F1<sub>char</sub> | 0.560 |
| Faithfulness (리스트 분기) | **0.000** |

**관찰.** Gold 의 "총 367명" 메타 표현이 list prompt 의 item 분할 로직에 혼입하여 F1<sub>strict</sub> 가 상대적으로 낮아짐. 또한 LLM이 반환한 3개 이름은 context 에 없어 Faithfulness 0. **본 벤치마크의 gold 포맷 이질성** 이 평가 상한을 제약함을 보여주는 사례로, Ch.Ⅶ 결론에서 future work "벤치마크 정제" 로 다룬다.

## CS-7.5 질의 2: 단일 학과 정답 (Simple, 최적 사례)

> **질의.** "최재원 교수의 소속 학과는?"
> **Gold.** `바이오의공학과`

| 프롬프트 | 응답 | F1<sub>strict</sub> | EM | Faith |
|---|---|---|---|---|
| List | `바이오의공학과` | **1.00** | **1.00** | 1.00 |

**관찰.** 단일 토큰 응답에서 list prompt 는 gold 와 완전히 일치. 본 benchmark 의 **Simple 타입 (40% 비중)** 에서 L-DWA 의 F1<sub>strict</sub> 가 0.906 에 도달하는 것은 이러한 단일/소규모 리스트 응답 패턴이 list prompt 의 sweet spot 이기 때문이다.

## CS-7.6 질의 10: 복수 학과 (Multi-hop, 부분 실패)

> **질의.** "배가 포함된 데이터사이언스 연구 프로젝트에 참여한 학과는?"
> **Gold.** `데이터사이언스학과, 무역학과`
> **List 프롬프트 응답.** `데이터사이언스학과` (1/2)

| 지표 | 값 |
|---|---|
| F1<sub>strict</sub> | 0.67 |
| F1<sub>substring</sub> | 0.50 |

**관찰.** Multi-hop 질의는 하나의 엔티티는 쉽게 찾으나 두 번째 이상의 엔티티 연결에서 누락 빈발. 이 유형에서 L-DWA 의 F1<sub>strict</sub> 는 0.365 (R-DWA 0.354 대비 +3%) 로 **향상 폭이 가장 작은** 영역이며, 본 논문의 **한계** 로 Ch.Ⅶ 에 명시.

## CS-7.7 종합 — List prompt 적용 전후 평가 구조 변화

| 관점 | Sentence 프롬프트 (기존) | List 프롬프트 (본 PR) |
|---|---|---|
| 출력 형식 | 자연어 문장 (filler 포함) | 콤마 구분 리스트 (항목만) |
| EM 가능성 | 구조적 0 (토큰 경계 불일치) | 0.39 (Simple 위주 회복) |
| F1<sub>strict</sub> 상한 | ≈ 0.2 (filler 잡음) | ≈ 0.87 (Simple 기준) |
| Faithfulness 엄격도 | 문장 any-token (관대) | 항목별 claim (엄격) |
| 지연 (sec) | 1.1 ~ 3.7 | 0.7 ~ 0.8 |
| 비용 (출력 토큰) | 평균 45 | 평균 12 |

**결론.** List prompt 는 *출력 형식과 gold 형식의 정렬* 이라는 단일 변경으로:

1. **F1<sub>strict</sub> 를 R-DWA 기준 +286% 개선** (0.137 → 0.529)
2. **EM 을 0 에서 0.39 로 복원**
3. **Faithfulness 엄격화** 로 환각 탐지 해상도 상승
4. **지연·비용 동시 절감** (출력 짧아짐)

이는 Triple-Hybrid RAG 의 **알고리즘 변경 없이** 획득한 개선이며, 본 논문의 평가 인프라가 **형식 민감도에 강건** 함을 입증한다.
