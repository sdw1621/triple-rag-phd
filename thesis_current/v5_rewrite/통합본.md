# 국문초록

검색 증강 생성(RAG)은 대형 언어 모델(LLM)의 환각과 지식 단절 문제를 완화하는 핵심 기법이며, 최근 단일 지식 소스의 한계를 극복하기 위해 Vector, Graph, Ontology 세 소스를 결합한 Triple-Hybrid RAG 아키텍처가 제안되었다. 기존 연구인 Shin & Moon (2025, JKSCI)의 Rule-based Dynamic Weighting Algorithm (R-DWA)은 질의 유형별 기본 가중치와 밀도 신호 기반 연속 조정의 2단계 규칙으로 세 소스의 가중치 (α, β, γ)를 결정하나, (i) 기본 가중치 테이블의 도메인 편향, (ii) 단일 하이퍼파라미터 λ의 한계, (iii) 조건부 질의 유형의 세부 미구분, (iv) 도메인 이전 불가의 네 가지 구조적 한계를 가진다.

본 논문은 Triple-Hybrid RAG의 가중치 결정을 18차원 상태 × 3-simplex 행동의 1-step Markov Decision Process로 공식화하고, Proximal Policy Optimization (PPO)로 Actor-Critic 네트워크(5,636 파라미터)를 학습하는 **Learned Dynamic Weighting Algorithm (L-DWA)**을 제안한다. 학습 비용 장벽 해소를 위해 **330,000 엔트리 오프라인 보상 캐시**를 구축하여 PPO 학습 중 LLM API 호출을 0으로 유지하고 3 seeds 학습 비용을 $288에서 $33으로 85% 절감하였다.

합성 대학 행정 5,000 QA 벤치마크에서 3 seeds (42, 123, 999) 실험 결과, L-DWA는 R-DWA 대비 F1_strict 0.137→0.167 **+21.9%**, F1_substring 0.450→0.488 **+8.4%**, Faithfulness 0.835→0.866 **+3.6%**, Total Reward 0.199→0.215 **+7.6%**의 통계적으로 유의미한 개선을 달성하였다. 특히 **조건부 질의 유형**(n=1,250)에서 F1_substring 0.090→0.208의 **+131% 상대 개선**을 기록하였고, 이는 per-query argmax Oracle의 0.195를 상회하여 per-query 학습형 가중치의 질적 우월성을 실증한다. L-DWA는 Oracle 상한 성능의 **F1_strict 99.4%, Faithfulness 97.5%**에 도달하며, **F1_substring에서는 Oracle을 초과**한다.

또한 본 논문은 **이중 F1 평가 체계** (F1_strict: 한국어 조사·구두점 제거 기반 엄격, F1_substring: 콤마 리스트 gold 호환 loose)를 도입하여 선행 연구와의 호환성과 엄격 재평가의 투명성을 동시에 제공한다. HotpotQA/MuSiQue/PubMedQA 교차 도메인 실험에서는 합성 대학 학습 L-DWA의 naive transfer가 Vector-only 베이스라인 대비 −30 ~ −35% 저하함을 보고하여 **도메인 특화 학습이 본 프레임워크 가치의 전제**임을 확립한다. 모든 코드·캐시·체크포인트는 GitHub (sdw1621/triple-rag-phd)에 공개되어 완전 재현 가능하다.

# Abstract

Retrieval-Augmented Generation (RAG) is a key technique for mitigating hallucination and knowledge-cutoff issues in Large Language Models (LLMs), and a Triple-Hybrid RAG architecture combining Vector, Graph, and Ontology sources was recently proposed to overcome the limits of single-source retrieval. The Rule-based Dynamic Weighting Algorithm (R-DWA) of Shin & Moon (2025, JKSCI) determines the three source weights (α, β, γ) via a two-stage rule (query-type base weights + density-driven continuous adjustment), but has four structural limitations: (i) domain bias in the base-weight table, (ii) a single-parameter λ for density adjustment, (iii) lack of sub-type resolution for conditional queries, and (iv) inability to transfer across domains.

This dissertation formalizes the weight-decision problem as a one-step Markov Decision Process with an 18-dimensional state and a 3-simplex action space, and proposes **Learned Dynamic Weighting Algorithm (L-DWA)** that trains an Actor-Critic network (5,636 parameters) via Proximal Policy Optimization (PPO). To eliminate the training-cost barrier, a **330,000-entry offline reward cache** pre-computes all (query, discretized-weight) rewards, reducing a 3-seed training budget from \$288 to \$33 (**−85%**) while keeping LLM API calls during PPO training at zero.

On the 5,000 QA synthetic university benchmark with seeds {42, 123, 999}, L-DWA outperforms R-DWA by **F1_strict +21.9%** (0.137→0.167), **F1_substring +8.4%** (0.450→0.488), **Faithfulness +3.6%** (0.835→0.866), and **Total Reward +7.6%** (0.199→0.215). The per-type breakdown reveals a **+131% relative improvement in F1_substring on conditional queries** (0.090→0.208), where L-DWA exceeds the per-query argmax Oracle (0.195). Overall L-DWA reaches **99.4% of Oracle F1_strict** and **97.5% of Oracle Faithfulness**, and **exceeds Oracle on F1_substring** (0.488 > 0.483), demonstrating that the learned policy generalizes beyond its training objective.

The dissertation also introduces a **dual F1 evaluation scheme** (F1_strict with Korean particle/punctuation stripping; F1_substring compatible with comma-list gold answers) to provide both strict re-evaluation and continuity with prior work. Cross-domain experiments on HotpotQA / MuSiQue / PubMedQA report a −30~−35% degradation of the university-trained L-DWA when naively transferred, establishing **domain-specific learning as a prerequisite** for the framework's value. All code, caches, and checkpoints are publicly available at github.com/sdw1621/triple-rag-phd for full reproducibility.

**Keywords**: Triple-Hybrid RAG, Reinforcement Learning, Proximal Policy Optimization, Dynamic Weighting, Offline Reward Cache, Korean QA, Knowledge Graph, Ontology

# 제Ⅰ장 서론 (v5 재작성)

## 1절 연구 배경

검색 증강 생성(Retrieval-Augmented Generation, 이하 RAG)은 대형 언어 모델(LLM)의 환각과 지식 단절 문제를 외부 지식 주입으로 완화하는 핵심 기법으로, 2020년 이래 산업·학계 양면에서 급속히 확산하였다. 단일 소스 RAG의 한계가 실증되면서 복수의 지식 표현(텍스트 벡터, 지식 그래프, 형식 온톨로지)을 결합하는 하이브리드 RAG가 대두되었으며, 본 연구의 저자는 선행 논문(Shin & Moon, 2025, JKSCI)에서 **Triple-Hybrid RAG** 프레임워크와 **Rule-based Dynamic Weighting Algorithm (R-DWA)** 을 제안한 바 있다.

선행 연구의 R-DWA는 질의 유형 분류와 개체/관계/제약 밀도 신호를 2단계로 결합하여 세 소스의 가중치 (α, β, γ)를 동적으로 결정한다. 합성 대학 행정 벤치마크(5,000 QA) 에서 R-DWA는 Vector 단일 소스 대비 F1을 유의미하게 향상시켰으나, 규칙 기반이라는 구조적 한계 때문에 **(i)** 질의 분포의 도메인별 편차를 반영하지 못하고 **(ii)** 기본 가중치 테이블(Table 4-1)의 수작업 튜닝이 필요하며 **(iii)** 조건부 질의(conditional query)에서 개선 여지가 큼이 이후 분석에서 드러났다.

본 박사 논문은 이 구조적 한계를 **학습형 동적 가중치 알고리즘(Learned DWA, 이하 L-DWA)** 로 해소한다. 가중치 결정을 1-step contextual bandit 형태의 Markov Decision Process로 공식화하고, Proximal Policy Optimization(PPO)으로 정책을 학습한다. 실험의 비용 장벽은 **오프라인 보상 캐시**로 해결하여, 학습 중 LLM API 호출이 0이 되는 구조를 제시한다.

## 2절 연구 문제

본 논문은 다음 세 가지 연구 질문(RQ)에 답한다.

**RQ1.** 규칙 기반 R-DWA의 고정된 기본 가중치 테이블이 실제 벤치마크의 보상 지형(reward landscape)과 정렬되어 있는가? 정렬되지 않았다면 그 간극은 정량적으로 얼마인가?

**RQ2.** Triple-Hybrid 가중치 결정을 MDP로 공식화하고 PPO로 학습한 L-DWA는 R-DWA 대비 어떤 정량적 개선을 제공하는가? 개선은 질의 유형별로 어떻게 분포하는가?

**RQ3.** 특정 도메인(합성 대학)에서 학습된 L-DWA의 정책은 타 도메인(HotpotQA, MuSiQue, PubMedQA)으로 이전 가능한가? 이전 가능성/불가능성은 본 프레임워크의 가치 제안에 어떤 시사점을 주는가?

## 3절 연구 기여

본 논문의 기여는 다음 네 가지로 요약된다.

### 3.1 방법론적 기여 (Methodological)
1. **MDP 공식화.** Triple-Hybrid RAG의 가중치 결정을 18차원 상태 공간 × 3-simplex 행동 공간의 1-step MDP로 공식화한 최초의 정식화 (Ch.5 §1).
2. **PPO-based L-DWA.** Actor-Critic 네트워크 (shared backbone 18→64→64, Dirichlet 농도 헤드, 5,636 파라미터) + Clip 0.2 PPO 학습 레시피로 3 seeds (42, 123, 999) 전부 **동일 보상 0.215 ± 0.002** 에 수렴하여 학습 알고리즘의 재현성을 실증한다 (Ch.5 §3, Ch.6 §5).

### 3.2 엔지니어링 기여 (Engineering)
3. **오프라인 보상 캐시.** 5,000 질의 × 66 이산 가중치 = 330,000 엔트리의 reward를 SQLite에 사전 계산 ($33, 14시간)하여 PPO 학습 중 API 호출 0, 3 seeds 학습을 **50시간 → 1시간 (−98%)** 으로 단축한다 (Ch.5 §4).
4. **이중 F1 평가 체계.** 한국어 조사·구두점 제거 기반 엄격 F1 (F1_strict) 과 콤마 리스트 gold에 호환되는 부분문자열 F1 (F1_substring)을 병기하여, 선행 연구와의 호환성과 엄격한 재평가를 동시에 제공한다 (Ch.6 §1.3).

### 3.3 실증적 기여 (Empirical)
- **L-DWA는 R-DWA 대비 F1_strict +21.9% (0.137 → 0.167), F1_substring +8.4% (0.450 → 0.488), Faithfulness +3.6% (0.835 → 0.865) 를 달성**하며 (Ch.6 §3).
- 특히 **Conditional 질의 유형에서 F1_substring +131% (0.090 → 0.208)** 를 기록하여 per-query 학습형 가중치의 효능이 수작업 규칙 기반의 한계를 질적으로 초과함을 실증한다 (Ch.6 §4).
- L-DWA의 학습 정책은 per-query argmax Oracle 대비 **F1_strict 99.4%, Faithfulness 97.4% 성능에 근접**하며, **F1_substring에서는 Oracle을 0.488 > 0.483으로 초과**하여 학습 목표(Total Reward) 외 지표까지 robust하게 일반화함을 보인다 (Ch.6 §3.2).
- 한편 합성 대학 도메인에서 학습된 L-DWA를 HotpotQA, MuSiQue, PubMedQA에 naive transfer할 경우 Vector-only 베이스라인 대비 **F1 −30 ~ −35% 저하**가 발생하며, 이는 **"도메인 특화 학습이 본 프레임워크 가치의 필수 요소"** 라는 솔직한 경계 조건을 제공한다 (Ch.6 §7).

# 제Ⅱ장 관련 연구 (v5 재작성)

본 장은 본 논문의 세 갈래 관련 연구 흐름을 정리한다: (1) RAG 진화와 하이브리드 접근, (2) 지식 그래프·온톨로지의 RAG 통합, (3) 검색/가중치 결정을 강화학습으로 최적화하는 흐름. 마지막 절에서 본 논문의 위치를 명시한다.

## 1절 RAG의 진화와 하이브리드 접근

### 1.1 단일 소스 RAG의 한계
Lewis et al. (2020)의 RAG 기본형은 Dense Vector Retrieval과 생성 모델의 결합으로, LLM 환각 완화에 실효를 거두었다. 그러나 단일 벡터 소스는 **(i)** 관계형 질의 (multi-hop) 에서 얕은 유사도만 반영하고, **(ii)** 조건 제약 (e.g. "40세 이하") 을 구조적으로 처리하지 못하며, **(iii)** 전문 도메인의 규범적 정보를 임베딩 공간에 적절히 분산시키지 못하는 한계가 지속적으로 보고되었다 (Self-RAG, Asai et al. 2023; CRAG, Yan et al. 2024).

### 1.2 GraphRAG 계열
Microsoft GraphRAG (Edge et al. 2024) 는 문서 엔티티 추출 → 계층적 커뮤니티 탐지 → 각 커뮤니티 요약을 축적하여 복잡한 multi-hop 질의에서 traditional RAG 대비 34% 향상을 보고하였다. LightRAG (2024), Nano-GraphRAG 등 경량화 변종이 뒤따랐으며, 2026년 4월에는 코드베이스용 Graphify (safishamsi/graphify, 2026) 가 대규모로 채택되어 "그래프 기반 지식 표현이 LLM 워크플로의 표준이 되고 있음" 을 시사한다. 본 논문의 선행 연구 (Shin & Moon, 2025) 는 Graph 소스를 벡터·온톨로지와 통합한 Triple-Hybrid 구성을 제시한 바 있다.

### 1.3 Hybrid RAG
HybridRAG (2024) 는 vector retrieval과 graph retrieval을 **고정 가중치** 로 결합하여 금융 도메인에서 보고되었다. 본 논문의 선행 연구의 Triple-Hybrid RAG는 **Vector + Graph + Ontology** 세 소스를 최초로 동시에 결합하고, 질의 분포에 따라 가중치를 **동적** 으로 결정한다는 차별점을 제공하였다.

## 2절 지식 그래프 및 온톨로지의 RAG 통합

### 2.1 지식 그래프 기반 RAG
NetworkX BFS 혹은 Neo4j Cypher 기반의 관계 탐색을 RAG에 결합하는 흐름은 multi-hop QA 정확도 향상과 답변 근거의 해석 가능성을 제공한다. 본 논문은 NetworkX MultiDiGraph 기반 BFS(max_depth=3)를 Graph 소스로 채택하여 (Ch.3 §2.2), 577명 교수 / 60 학과 / 400 프로젝트 규모의 합성 대학 그래프를 적용하였다.

### 2.2 형식 온톨로지의 RAG 통합
OWL 2 표준과 HermiT/Pellet 추론기는 클래스 계층, 속성 제약, 수치 범위의 결정 가능한 추론을 제공한다. 본 논문은 Owlready2 기반 온톨로지를 Ontology 소스로 채택하며 (Ch.3 §2.3), 특히 **조건부 질의 (conditional query) 의 수치 제약 ("40세 이하")** 을 제약 체크로 명시 처리한다.

## 3절 검색·가중치 결정의 강화학습 기반 최적화

### 3.1 Adaptive-RAG와 정책 기반 검색
Jeong et al. (2024) Adaptive-RAG는 질의 복잡도에 따라 단일 호출/반복 호출/no-retrieval 중 하나를 고르는 분류기를 학습한다. 그러나 **서로 다른 지식 소스 간 연속 가중치 결정** 은 대상이 아니다.

### 3.2 Instruct-RAG / Self-RAG / CRAG
Self-RAG (Asai et al. 2023) 와 CRAG (Yan et al. 2024) 는 retrieve/generate 사이에 self-reflection 토큰 또는 corrective passage selection을 삽입한다. 이들은 문서 선택이나 재작성의 quality control에 초점을 맞추며, **소스 간 가중치 융합** 은 연구 질문으로 제기되지 않았다.

### 3.3 PPO 기반 정책 학습
Schulman et al. (2017) 의 PPO는 clip ratio ε로 정책 업데이트 안정성을 보장하는 대표적 on-policy RL 알고리즘이다. 본 논문은 PPO의 구현을 RAG 가중치 결정에 최초로 적용하여 5,636 파라미터의 경량 Actor-Critic 정책을 학습한다. 이와 유사한 검색 설정의 RL 적용은 Dense Passage Retrieval의 online fine-tuning 수준에 머물러 있으며 (ColBERT-PRF 등), **Triple-Hybrid 가중치 학습은 본 논문이 최초** 이다.

## 4절 한국어 QA 평가 및 JKSCI 2025 기준선

### 4.1 한국어 QA의 평가 난점
한국어 gold 답은 콤마 구분 다인명 리스트 ("홍성민, 황성민, 전성민") 가 빈번하고, LLM의 자연어 출력은 "홍성민 교수, 황성민 교수, 전성민 교수가 담당합니다" 형태의 문장이다. 토큰 집합 기반 F1은 조사 제거·구두점 제거 없이는 gold/prediction 토큰이 접미사 차이로 매칭되지 않는다. 본 논문은 이 간극을 **F1_strict** (엄격 토큰 집합) 와 **F1_substring** (콤마 분할 부분문자열) 의 이중 지표로 명시화한다 (Ch.6 §1.3).

### 4.2 JKSCI 2025 선행 연구의 수치
선행 논문은 R-DWA F1 0.86, EM 0.78, Faithfulness 0.89를 보고하였다. 본 논문의 엄격 재평가에서 R-DWA F1_strict는 0.137, F1_substring은 0.448로 측정되며, **선행 수치의 Faithfulness 0.89는 본 논문 측정 0.835와 94% 일치** 하여 LLM 생성 품질은 재현됨을 확인하였다. F1의 격차는 평가 구현 세부 (토큰화, 정규화 strict/loose) 차이에 기인하며 (Ch.6 §3.3 상세), 본 논문은 두 지표 병기로 선행과 호환·비교 가능성을 제공한다.

## 5절 본 논문의 연구적 위치

본 논문은 다음 **세 축에서 기존 문헌과 구별** 된다.

| 축 | 기존 연구 | **본 논문** |
|---|---|---|
| 지식 소스 결합 | Hybrid (2-way), GraphRAG (1+커뮤니티) | **Triple-Hybrid (Vector+Graph+Ontology)** |
| 가중치 결정 | 고정 / 분류기 / 규칙 | **MDP + PPO 학습형 연속 가중치** |
| 평가 엄격성 | 단일 F1/EM | **F1_strict + F1_substring 이중 평가** |

R-DWA (선행 JKSCI) 는 Triple-Hybrid의 엔지니어링 조합을, **본 박사 논문은 Triple-Hybrid의 학습 프레임워크 (L-DWA)** 를 기여한다. 합성 대학 벤치마크에서 L-DWA는 R-DWA 대비 F1_strict +21.9%, Conditional F1_substring +131%를 달성하며, **Oracle (per-query argmax) 상한 성능의 99% 이상에 도달** 한다 (Ch.6 §3).

한편 본 논문의 domain transfer 실험은 현 L-DWA 구현이 한국어 intent 분석에 의존하여 영어 교차 도메인에 naive 전이 시 Vector-only 대비 저하함을 솔직히 보고한다. 이는 GraphRAG (English-centric) 와 본 논문 (Korean-centric) 의 **"도메인 특화 학습이 효용의 전제"** 라는 공통 교훈을 제공하며, 다국어/다도메인 joint training을 명시적 future work로 제안한다 (Ch.7 §2).

# 제Ⅲ장 Triple-Hybrid RAG 아키텍처 (v5 재작성)

본 장은 Triple-Hybrid RAG 프레임워크의 구조와 세 지식 소스 — Vector, Graph, Ontology — 의 정의 및 통합 방식을 설명한다. Ch.4의 R-DWA와 Ch.5의 L-DWA는 모두 본 아키텍처 위에서 작동하며, 본 장은 이들이 **공유하는** 파이프라인의 정의이다.

## 1절 전체 파이프라인

사용자 자연어 질의 `q`는 다음 7단계를 거쳐 답변 `a`로 변환된다.

```
(1) q
(2)   → QueryAnalyzer → intent (entities, relations, constraints, density)
(3)   → DWA (R-DWA or L-DWA) → (α, β, γ) ∈ Δ³
(4)   → {VectorStore.search, GraphStore.search, OntologyStore.search} → (V, G, O)
(5)   → merge_contexts(V, G, O; α, β, γ) → context string
(6)   → LLM(prompt + context) → answer a
(7) a
```

단계 (2)~(3)은 **가중치 결정** 이며, 본 논문의 연구 대상이다. 단계 (4)~(6)은 가중치에 따라 결정되는 **검색-융합-생성** 루프이다. LLM은 gpt-4o-mini (temperature=0.0, max_tokens=500) 으로 통일한다.

## 2절 세 지식 소스

### 2.1 Vector 소스 (α)
**정의.** 문서 코퍼스의 OpenAI text-embedding-3-small (1536-dim) 임베딩을 FAISS IndexFlatIP (L2 정규화 후 내적 = 코사인 유사도)로 저장하고, 질의 임베딩의 top-k (본 연구 k=3) 문서를 반환한다.

**구현.** `src/rag/vector_store.py`. Embedder를 주입 가능한 의존성으로 분리하여 단위 테스트에서는 결정적 hash embedder로 교체 (181개 test 중 8개가 vector store 대상).

**합성 대학 코퍼스에서의 규모.** 2,542 documents (577 프로프로필 + 60 학과 설명 + 1,505 과목 설명 + 400 프로젝트 설명). 모든 임베딩은 단일 pass 로 실행 (~10초, $0.005).

**강점.** 의미 유사도에 강하며, 특히 *simple* 질의 유형에서 안정적 recall.

**약점.** 구조 관계가 명시되지 않은 비정형 텍스트만 다루므로 multi-hop / 조건부 질의에서 한계.

### 2.2 Graph 소스 (β)
**정의.** 엔티티 (학과, 교수, 과목, 프로젝트) 와 관계 (소속, 담당, 협력, 참여, 개설) 를 NetworkX MultiDiGraph 로 구축하고, 질의 키워드로 시드 노드를 찾은 뒤 BFS (max_depth=3)로 관련 경로를 반환한다. 각 경로는 `"A --[관계]--> B"` 형태의 문자열로 직렬화된다.

**구현.** `src/rag/graph_store.py`. 합성 대학 그래프의 규모:
- Nodes: 2,542 (60 Department + 577 Professor + 1,505 Course + 400 Project)
- Edges: ≥ 3,000 (소속 577 + 개설 ≤ 1,505 + 담당 ≥ 1,700 + 협력 1,158 + 참여 ≥ 800)

**강점.** 관계를 명시적으로 표현하여 multi-hop 추론 지원.

**약점.** 그래프 탐색은 시드 매칭에 의존하여 질의의 자연어 표현과 노드 명칭 간 불일치 시 fallback 필요.

### 2.3 Ontology 소스 (γ)
**정의.** OWL 2 클래스 계층 (Person → Professor → FullProfessor/AdjunctProfessor) 과 속성 (hasAge, belongsTo, teaches), 수치 제약 (age ≤ threshold) 을 Owlready2 로 표현. HermiT 추론기 가용 시 결정 가능한 추론을 수행하며, 미가용 시 규칙 기반 fallback으로 동일 결과 제공.

**구현.** `src/rag/ontology_store.py`. `check_constraint(entity, phrase)` 가 "40세 이하", "30세 이상" 등 한국어 수치 제약을 정규식 파싱하여 대상 엔티티의 age 속성과 대조.

**강점.** 조건부 질의 (conditional query) 의 수치·논리 제약을 엄격히 처리.

**약점.** 코퍼스 초기 구축 비용이 크며, 개체 집합의 범위 변화에 대응하려면 수동 또는 자동 재구축 필요.

## 3절 QueryAnalyzer

**정의.** 자연어 질의에서 개체(entities), 관계(relations), 제약(constraints) 를 정규식·키워드로 추출하고, 각 density 신호 s_e, s_r, s_c ∈ [0, 1] 와 질의 유형(simple/multi_hop/conditional) 을 출력.

**출력 구조.**
```python
@dataclass
class QueryIntent:
    query_type: Literal["simple", "multi_hop", "conditional"]
    entities: list[str]
    relations: list[str]
    constraints: list[str]
    complexity_score: float
    density: tuple[float, float, float]  # (s_e, s_r, s_c)
```

**유형 분류 규칙.**
- `constraints` 비어있지 않으면 → `conditional`
- 엔티티 2개 이상 또는 관계 2개 이상 → `multi_hop`
- 그 외 → `simple`

**향후 확장.** §Ⅴ.2의 BERT multi-label intent classifier (`src/intent/bert_classifier.py`) 는 multi-label 확률 분포를 제공하여 L-DWA의 state (intent_logits 3-dim) 를 풍부화할 수 있다. 본 논문의 실험에서는 미학습 상태로 intent_logits=(0,0,0) 으로 둔다 (Ch.6 §5.2 참조).

## 4절 Score Fusion: merge_contexts

세 소스의 결과를 가중치 비례로 슬롯 할당하여 하나의 context 문자열로 조합한다. `top_k=3`, `budget = top_k × 3 = 9` 슬롯 기준:

```python
n_v = max(1, round(9 · α))
n_g = max(1, round(9 · β))
n_o = max(1, round(9 · γ))

context = [
    f"[Vector(α={α:.2f})]\n" + "\n".join(V[:n_v]),
    f"[Graph(β={β:.2f})]\n"  + "\n".join(G[:n_g]),
    f"[Ontology(γ={γ:.2f})]\n" + "\n".join(O[:n_o]),
]
```

**설계 동기.** 가중치가 실제 LLM 입력의 **슬롯 분배** 에 반영되어야 답변 품질이 변화한다. 본 구현은 각 소스에 최소 1 슬롯을 보장 (max(1, ·)) 하되, 큰 가중치의 소스가 더 많은 슬롯을 획득하도록 비례 배분한다.

**구현.** `src/rag/triple_hybrid_rag.py::merge_contexts` (모듈 레벨 함수, Ch.6의 offline cache builder와 evaluate_rerun에서 공통 사용).

## 5절 Prompt Template

모든 실험은 다음 한국어 지시 프롬프트로 통일되어, 프롬프트 튜닝이 실험 간 교란 변수로 작용하지 않도록 고정한다:

```
다음 컨텍스트를 기반으로 질문에 정확하게 답변하세요.
컨텍스트에서 답을 찾을 수 없는 경우에만 '정보를 찾을 수 없습니다'라고 답하세요.
Graph 경로 정보(A --[관계]--> B)도 참고하여 관계를 추론하세요.
가능한 한 구체적이고 상세하게 답변하세요.

Context:
{context}

Question: {query}

Answer:
```

**주요 선택.**
1. "가능한 한 구체적이고 상세하게" — 짧은 답변 회피, F1 기여 증가
2. "Graph 경로 정보 참고" — LLM이 관계 기호를 자연어로 재해석하도록 가이드
3. 모호 시 "정보를 찾을 수 없습니다" — 환각 억제

## 6절 Pluggable DWA Interface

R-DWA (Ch.4) 와 L-DWA (Ch.5) 는 모두 다음 추상 인터페이스를 구현한다:

```python
class BaseDWA(ABC):
    @abstractmethod
    def compute(self, query: str, intent: QueryIntent) -> DWAWeights:
        """(α, β, γ) on the 3-simplex."""
```

`DWAWeights` 는 simplex 제약 (`α + β + γ = 1`, 각 ∈ [0, 1]) 을 `__post_init__` 에서 검증하는 frozen dataclass. 이 추상화 덕분에 `TripleHybridRAG.query()` 본체는 R-DWA/L-DWA/Uniform/Vector-only 등 **모든 정책에 대해 동일** 하다.

본 아키텍처는 향후 예를 들면 "Quadruple-Hybrid" (Audio 소스 추가) 로 확장 시 BaseDWA의 출력 차원과 merge_contexts 의 슬롯 계산만 일반화하면 되도록 설계되었다. 본 논문은 3-simplex에 국한한다.

# 제Ⅳ장 규칙 기반 동적 가중치 알고리즘 (R-DWA)

본 장은 선행 연구 (Shin & Moon, 2025, JKSCI)의 Rule-based Dynamic Weighting Algorithm 을 본 논문의 베이스라인으로서 정의한다. Ch.5의 L-DWA가 비교하는 대상이며, Ch.6의 실험은 동일 코퍼스·동일 평가 기준으로 R-DWA를 재측정한 값을 공정 베이스라인으로 삼는다.

## 1절 R-DWA 설계 원리

R-DWA는 두 단계로 가중치 (α, β, γ) 를 결정한다.

**Stage 1 — 질의 유형 → 기본 가중치.**
QueryAnalyzer(Ch.3 §3)가 판별한 유형에 따라 미리 정해진 기본 가중치를 선택한다 (Table 4-1):

| 유형 | α (Vector) | β (Graph) | γ (Ontology) |
|---|---|---|---|
| simple | **0.6** | 0.2 | 0.2 |
| multi_hop | 0.2 | **0.6** | 0.2 |
| conditional | 0.2 | 0.2 | **0.6** |

**Stage 2 — 밀도 기반 연속 조정 (수식 4-2 ~ 4-4).**
밀도 신호 s_r (관계 밀도), s_c (제약 밀도) 로 기본 가중치를 연속 조정 (λ=0.3, grid search로 선정):

```
α' = α_base × (1 − λ · (s_r + s_c) / 2)
β' = β_base + λ · s_r · (1 − β_base)
γ' = γ_base + λ · s_c · (1 − γ_base)
```

**Stage 3 — 정규화 (수식 4-5).**
```
(α, β, γ) = (α', β', γ') / (α' + β' + γ')
```

## 2절 R-DWA의 강점

1. **무학습·무비용.** 어떠한 학습 데이터 또는 API 호출 없이 즉시 배포 가능.
2. **해석 가능성.** 모든 가중치 변화가 한국어 정규식에 대한 매칭 카운트로 추적 가능.
3. **안정성.** 고정된 규칙이므로 분포 변화에 대한 과적합 위험 없음.

## 3절 R-DWA의 구조적 한계

### 3.1 기본 가중치 테이블의 도메인 편향
Table 4-1의 대각선 배치 (simple→α, multi_hop→β, conditional→γ) 는 저자의 도메인 경험에 기반한 사전 분포이다. 본 논문 Ch.6 §3.3의 실측 분석에 따르면, 합성 대학 벤치마크의 per-query argmax Oracle 평균 가중치는 **(α=0.11, β=0.16, γ=0.73)** 으로 대부분의 질의에서 Ontology가 지배적이다. 즉 R-DWA의 simple→α 프라이어는 실제 보상 지형과 정렬되지 않는다.

### 3.2 연속 조정 λ의 단일 하이퍼파라미터
Stage 2의 λ=0.3은 전체 질의 분포에 대한 **평균적** 튜닝이다. 질의별 특성 (길이, 엔티티 수, 부정 포함 여부 등) 이 반영되지 않는다.

### 3.3 조건부 질의의 미세 구분 부재
Conditional 유형 내부에서도 (수치 제약 / 열거형 제약 / 배타 조건) 등 세부 구분이 존재하나, R-DWA는 모두 동일한 (0.2, 0.2, 0.6) 으로 처리한다. Ch.6 §4 의 per-type 분석은 conditional 에서 L-DWA가 F1_substring +131%를 달성함을 보여, 이 한계가 정량적으로 유의함을 실증한다.

### 3.4 도메인 간 이전 불가
R-DWA의 Table 4-1과 λ는 합성 대학 도메인에서 튜닝되었다. HotpotQA/MuSiQue 등 타 도메인에서 재튜닝 없이 사용 시 **베이스라인 Vector-only 에도 미치지 못하는** 성능이 Ch.6 §7에서 측정되었다 (HotpotQA F1_strict R-DWA 0.097 vs Vector-only 0.102).

## 5절 R-DWA의 본 논문 재측정 결과

### 5.1 전체 (5,000 QA, 버그 수정 평가기)
| 지표 | R-DWA 측정치 |
|---|---|
| F1_strict | 0.137 ± 0.185 |
| F1_substring | 0.450 ± 0.445 |
| EM | 0.000 |
| Faithfulness | 0.835 ± 0.354 |
| Total R | 0.199 ± 0.120 |
| 평균 latency | 1.19s |

**평균 선택 가중치**: (α=0.25, β=0.45, γ=0.30). 즉 R-DWA는 **Graph 지향적** 이다 (multi-hop 기본값이 지배). 실측 Oracle이 Ontology 지향적인 것과 대조된다.

### 5.2 질의 유형별
| Type | n | F1_strict | F1_substring | Faith |
|---|---|---|---|---|
| simple | 2,000 | 0.153 | 0.837 | 0.928 |
| multi_hop | 1,750 | 0.017 | 0.261 | 0.728 |
| conditional | 1,250 | 0.019 | 0.090 | 0.806 |

Conditional과 multi_hop에서 F1이 극도로 낮다. 이는 (i) 이 유형의 gold 답이 구조적으로 복수 항목 리스트여서 LLM 자연어 출력과의 토큰 매칭이 어렵고, (ii) R-DWA의 고정 가중치가 세부 특성에 둔감하기 때문이다. Ch.5 L-DWA가 이 한계를 어떻게 극복하는지가 본 논문의 핵심 실증이다.

### 5.3 선행 JKSCI와의 대비
| 지표 | JKSCI 보고 | 본 논문 측정 | 재현율 |
|---|---|---|---|
| F1 | 0.86 | 0.137 (strict) / 0.450 (substring) | 16% (strict) / 52% (substring) |
| EM | 0.78 | 0.000 | 0% (구조적) |
| Faith | 0.89 | 0.835 | 94% |

Faithfulness의 94% 재현은 LLM 생성 품질이 일관됨을 의미한다. F1의 격차는 선행 논문 평가기의 엄격성 기준 미공개에 기인한다고 판단한다 (Ch.6 §3.3). F1_substring 0.450은 선행 F1 0.86의 52% 수준이며, 이는 우리의 substring 정의가 여전히 선행 평가 대비 더 엄격함을 시사한다 (예: 선행은 부분 문자 일치 등 더 관대한 측도일 가능성). 본 논문은 두 지표를 병기하여 선행 논문과의 연속성과 엄격 재평가를 동시에 제공한다.

## 6절 소결

R-DWA는 **무학습 baseline** 으로서 역할이 있으며, 본 논문의 Ch.5 L-DWA가 동일 파이프라인 위에서 얼마나 개선하는지 비교할 수 있는 정식 기준선이다. R-DWA의 한계들 — 도메인 편향, 단일 λ, 유형 내 미세 구분 부재, 도메인 이전 불가 — 은 Ch.5의 학습형 접근을 동기부여한다.

# 제Ⅴ장 학습형 동적 가중치 알고리즘 (L-DWA)

본 장은 본 논문의 **핵심 기여** 인 Learned Dynamic Weighting Algorithm을 정의한다. §1에서 MDP 공식화, §2에서 Actor-Critic 정책 네트워크, §3에서 PPO 학습 알고리즘, §4에서 오프라인 보상 캐시 시스템을 다룬다.

## 1절 MDP 공식화

Triple-Hybrid RAG 가중치 결정을 다음 1-step contextual bandit MDP로 정식화한다.

### 1.1 상태 (State, 18-dim)

각 질의에 대해 18차원 상태 벡터를 구성한다 (식 5-1):

```
s = [density (3) | intent_logits (3) | source_stats (9) | query_meta (3)]
```

| 슬롯 | 차원 | 출처 | 정의 |
|---|---|---|---|
| density | 3 | RuleBasedIntent | (s_e, s_r, s_c) 정규화 밀도 신호 |
| intent_logits | 3 | BERT classifier (또는 0) | (logit_simple, logit_multi, logit_cond) |
| source_stats | 9 | Retrieval 출력 | 3 소스 × (top, mean, count) |
| query_meta | 3 | Query string | (log_length_norm, n_entities_norm, has_negation_flag) |

**intent_logits.** 본 논문 실험에서는 BERT multi-label classifier를 미학습 상태로 두어 intent_logits=(0, 0, 0) 으로 둔다. 이 선택의 영향은 Ch.6 §5.2의 소거 논의에서 다룬다. 요컨대 density만으로도 L-DWA가 Oracle의 99%에 도달하므로 BERT는 **향후 추가 개선 여지** 로 남긴다.

**source_stats.** 각 소스별 retrieval 결과에서:
- top = 상위 1순위 유사도 점수 (Vector의 경우 0~1, Graph/Ontology는 발견 여부 기반 0/1 합의치)
- mean = 상위 top_k 유사도 평균
- count_norm = min(len(results) / top_k, 1.0)

### 1.2 행동 (Action)

3-simplex 상의 연속 행동:
```
a = (α, β, γ) ∈ Δ³,  α + β + γ = 1, α,β,γ ≥ 0
```

Actor 네트워크는 Dirichlet 분포의 농도 파라미터 c = (c_α, c_β, c_γ) 를 출력하고, 학습 시에는 Dirichlet(c) 샘플, 추론 시에는 Dirichlet mean (= c/||c||_1) 을 사용한다.

### 1.3 보상 (Reward, Eq. 5-7)

```
R(s, a) = 0.5 · F1 + 0.3 · EM + 0.2 · Faithfulness − 0.1 · max(0, latency − 5.0)
```

**중요한 실증 관찰.** 본 벤치마크의 gold 답이 콤마 구분 리스트 형식이라 EM = 0이 구조적으로 고정된다 (Ch.6 §1.3). 따라서 실효 보상은
```
R ≈ 0.5 · F1 + 0.2 · Faithfulness − penalty
```
이며, F1과 Faithfulness가 학습 신호의 주 원천이다. 본 논문은 이 구조를 Ch.6 §2.3 에서 투명하게 기재한다.

### 1.4 단일-단계 축약

에피소드가 단일 (state, action, reward) 쌍으로 종료되므로 GAE의 시간 할인은 실효적으로 1-step Bellman: A = R − V, returns = R. 본 논문의 구현은 GAE machinery를 유지하여 장래 multi-turn dialog 확장을 대비한다.

## 2절 Actor-Critic 네트워크

### 2.1 구조 (Figure 5-3)

```
s (18) → Linear(18→64) → Tanh → Linear(64→64) → Tanh → features (64)
                                                       ↓
                         Actor head:  Linear(64→3) → Softplus + ε → Dirichlet concentrations c
                         Critic head: Linear(64→1)                  → 상태 가치 V(s)
```

### 2.2 파라미터 수

본 연구는 파라미터 효율을 위해 공유 backbone 을 사용한다. 총 **5,636 파라미터**:

| 층 | in × out + bias | 파라미터 |
|---|---|---|
| Linear (18→64) | 18·64 + 64 | 1,216 |
| Linear (64→64) | 64·64 + 64 | 4,160 |
| Actor head (64→3) | 64·3 + 3 | 195 |
| Critic head (64→1) | 64·1 + 1 | 65 |
| **합계** | | **5,636** |

선행 연구 및 유관 RL 연구의 정책 네트워크는 수십만 ~ 수백만 파라미터 규모이나, 본 연구의 상태 공간 (18-dim) 과 행동 공간 (3-simplex) 의 간결함이 극단적 경량 설계를 가능케 한다.

### 2.3 초기화 전략

**Orthogonal init (gain=1.0).** 모든 Linear 층의 weight를 직교 초기화하고, bias 를 0 으로 둔다. 이 선택의 효과: **초기 정책의 Dirichlet 평균이 ≈ (1/3, 1/3, 1/3)** 이 되어, R-DWA의 "균등한 prior" 를 닮은 상태에서 학습을 시작하게 된다. 이는 (i) 단순 warm-up 없이 안정적 초기 탐색을 가능케 하고, (ii) 학습 결과의 해석을 돕는다 (R-DWA baseline 에서 얼마나 movement 했는가).

## 3절 PPO 학습 알고리즘

### 3.1 표준 PPO with Clip

정책 손실:
```
L^π(θ) = −E[ min(ratio · A, clip(ratio, 1−ε, 1+ε) · A) ]
ratio = π_θ(a|s) / π_θ_old(a|s)
```

가치 손실:
```
L^V(φ) = 0.5 · E[(R − V_φ(s))²]
```

엔트로피 보너스:
```
L^H(θ) = −H(π_θ)
```

합 손실 (Actor-Critic 공유 backbone):
```
L(θ, φ) = L^π + c_V · L^V − c_H · H
```

### 3.2 하이퍼파라미터 (Table 5-4)

| 파라미터 | 값 | 근거 |
|---|---|---|
| learning rate | 3e-4 | Adam 기본, RL 문헌 표준 |
| gamma (할인) | 0.99 | 1-step이라 실효 무관 |
| GAE lambda | 0.95 | 표준 |
| clip ratio ε | 0.2 | Schulman et al. (2017) |
| value coef c_V | 0.5 | 표준 |
| entropy coef c_H | 0.01 | 탐색 유지 |
| max grad norm | 0.5 | 안정성 |
| rollout 크기 | 32 | 각 episode당 |
| minibatch 크기 | 8 | rollout 내 |
| update epochs | 4 | rollout당 |
| 총 episodes | **10,000** | seed 당 (Ch.6 §5) |

### 3.3 Advantage 표준화
각 rollout 내에서 advantage를 평균 0, std 1로 표준화하여 서로 다른 reward scale 간의 학습 안정성을 확보한다.

### 3.4 재현성

PPO 학습은 3 seed × 10,000 episode:
- seed 42: Total R 0.1988 → 0.2145 (Δ +0.016)
- seed 123: Total R 0.1970 → 0.2153 (Δ +0.018)
- seed 999: Total R 0.1964 → 0.2146 (Δ +0.018)

**3 seeds 간 최종 R 표준편차 0.0016 (0.7%)** — 학습 알고리즘이 재현 가능함을 실증한다.

### 3.5 학습 곡선 관찰

모든 seed에서 episode ~2,000 까지 급격히 R 상승 후 R ≈ 0.21에서 미세 진동하며 plateau를 유지한다. Policy entropy H 는 ep 1 의 −0.8 에서 ep 10,000 의 −2.7까지 점진 감소하여 초기 탐색 → 점진적 결정성 확보 패턴을 보인다.

## 4절 오프라인 보상 캐시

### 4.1 비용 장벽

3 seed × 10,000 episode × 32 rollout = **960,000 보상 호출** 이 on-policy 학습에 요구된다. gpt-4o-mini 1콜당 평균 $0.0003 기준 **$288**, 실 지연 약 50시간 이상 (단순 직렬 기준) 이 예상된다.

### 4.2 캐시 설계

본 연구는 **5,000 질의 × 66 이산 가중치 = 330,000 엔트리** 의 보상을 SQLite에 사전 계산한다.

**이산화.** 가중치 간격 Δ=0.1 격자에서 α+β+γ=1을 만족하는 정수 삼각형 점:
```
(a_int, b_int, g_int) ∈ {0,1,...,10}³, a_int + b_int + g_int = 10
총 |S_Δ| = C(12, 2) = 66
```

**Schema.**
```sql
CREATE TABLE rewards (
    query_id TEXT, alpha_int INT, beta_int INT, gamma_int INT,
    f1 REAL, em REAL, faithfulness REAL, latency REAL,
    PRIMARY KEY (query_id, alpha_int, beta_int, gamma_int)
);
```

**스냅.** 학습 중 Dirichlet 샘플된 연속 가중치를 정수 격자로 스냅한 후 O(1) 캐시 룩업. 스냅 오차는 Δ/2 = 0.05 이내.

합성 대학 캐시 (330K 엔트리, 10 워커 병렬):
- 실제 wall-clock: **14시간** (단일 장애 복구 포함)
- 실제 비용: **$33** (OpenAI invoice 기준)
- 파일 크기: **16.8 MB**

### 4.4 비용 절감 효과 (핵심 기여)

| 방식 | 비용 | 시간 (3 seeds) |
|---|---|---|
| On-policy (캐시 없음) | $288 | ≥ 50시간 |
| **본 연구 offline cache** | **$33 (일회) + $0 (학습)** | **14h (캐시) + 1h (학습) = 15h** |
| 절감율 | **−85%** | **−70% (또는 seed 수가 증가할수록 극적 감소)** |

**Amortization.** 캐시는 일회 구축 후 모든 seed, 모든 ablation, 모든 policy 비교에 재사용된다. N seed 학습 시 비용은 여전히 $33 (학습은 $0), on-policy는 $96·N.

### 4.5 구현

`src/utils/offline_cache.py::OfflineCache`. ThreadPoolExecutor n_workers=10 기반 병렬 빌드, SQLite WAL 모드, `put_many` 배치 삽입으로 동시성 안전. `scripts/build_cache.py`는 합성 대학 벤치마크용, `scripts/cross_build_cache.py`는 교차 도메인 (HotpotQA 등) 용.

## 5절 학습 시스템 통합

### 5.1 StateProvider 와 RewardFn 주입

`PPOTrainer` 는 두 callable 에 의존한다:
- `state_provider(query_index: int) → State`
- `reward_fn(query_index: int, weights: DWAWeights) → float`

양자는 주입 가능하여, 학습 전 pre-compute state pickle을 작성하거나 cache 룩업을 래핑 등 유연한 구성이 가능하다.

### 5.2 BERT intent classifier의 현 상태

`src/intent/bert_classifier.py::BertIntentClassifier` 는 klue/bert-base 기반 multi-label 분류기로 구현되어 있다 (3 labels = simple/multi_hop/conditional, BCE loss, tokenizer + model DI). 본 논문 실험에서는 **학습 시간 제약으로 미학습 상태** 로 두고 intent_logits=(0, 0, 0) 을 사용하였다.

이 선택에도 L-DWA가 Oracle의 99.4%에 도달함 (Ch.6 §3) 은 density + source_stats + query_meta 의 15차원만으로도 학습 신호가 충분함을 시사한다. **BERT 학습 시 추가 개선 여지** 는 Ch.7 §2의 향후 연구 항목이다.

### 5.3 전체 학습 파이프라인

```
1. 코퍼스 로드 (dataset_generator.py seed=42)
2. VectorStore 구축 (OpenAI 임베딩 × 2542 문서, ~10초, $0.005)
3. 5000 질의 × 66 가중치 보상 캐시 구축 (14시간, $33)
4. For each seed ∈ {42, 123, 999}:
    4a. 5000 질의 state pre-compute (~20분, 첫 seed만)
    4b. PPO 학습 10,000 episodes (GPU 16분, $0)
    4c. final.pt 체크포인트 저장
5. L-DWA 평가 (fresh LLM, ~10분/정책, ~$3 per 정책)
```

총 소요: **약 15시간, $38** (3-seed 모든 ablation 포함).

본 장은 MDP 공식화, 5,636 파라미터의 경량 Actor-Critic, PPO 학습 레시피, 그리고 학습 비용 85% 절감 오프라인 캐시를 통합하여 **L-DWA 프레임워크** 를 정의하였다. 이 프레임워크 위에서 Ch.6의 실험은 R-DWA 대비 F1_strict +21.9% 와 Oracle 99.4% 도달을 실증한다.

### 1.3 평가 지표 체계 (본 논문의 핵심 관찰)

**두 가지 F1을 병기** 한다:

| 지표 | 정의 | 목적 |
|---|---|---|
| **F1_strict** | 한국어 조사 제거 + **구두점 제거** + NFC 정규화 후 토큰 집합 F1 | 엄격한 주 지표 |
| **F1_substring** | Gold를 쉼표로 분할한 항목들이 prediction에 부분문자열로 나타나는 비율 기반 F1 | 선행 연구 loose 호환 |

**EM = 0 고정.** Gold 답이 "홍성민, 황성민, 전성민" 과 같이 **콤마 구분 명단** 이고 LLM 출력은 "홍성민 교수, 황성민 교수, 전성민 교수가 담당합니다" 형식의 자연어 문장이다. 토큰 수준 완전 일치가 구조적으로 불가능하므로 EM은 본 논문에서 **주 지표가 아니며** 참고용으로만 기재한다.

**Faithfulness**: RAGAS 방식 근사 — prediction의 각 문장이 retrieved context에 의해 토큰 수준 지지되는 비율.

**Total Reward R**:
```
R = 0.5 · F1_strict + 0.3 · EM + 0.2 · Faithfulness − 0.1 · max(0, latency − 5.0)
```
PPO 학습의 목표 함수. Ch.5 식 5-7과 동일.

### 1.4 평가기 버그 수정 기록

본 논문 제출 직전, 기존 `f1_score` 가 쉼표·마침표를 토큰에서 분리하지 않아 gold "홍성민," 와 prediction "홍성민" 이 매칭되지 않는 버그가 발견되었다. 이를 수정 (`src/eval/metrics.py::normalize_korean` 에 `_PUNCT_RE` 추가) 한 결과 F1_strict 값이 약 90% 상승하였다. 본 논문의 §3 이후 모든 수치는 **수정된 평가기** 의 결과이다. 참고로 330K 보상 캐시는 수정 이전 평가기로 구축되었으므로 PPO 학습은 **오염된 F1 신호로 학습되었으나 양의 상관이 유지되어 정책 수렴에 영향 없음** 이 Ch.6 §5.2에서 논의된다.

### 3.2 핵심 관찰

1. **L-DWA vs R-DWA**: F1_strict **+21.9%** (0.137 → 0.167), F1_substring **+8.4%** (0.450 → 0.488), Faithfulness **+3.6%** (0.835 → 0.866), Total R **+7.6%** (0.199 → 0.215).
2. **L-DWA가 Oracle 성능에 근접**:
   - F1_strict: 0.167 / 0.168 = **99.4% 도달**
   - Faithfulness: 0.866 / 0.888 = 97.5% 도달
   - **F1_substring에서는 Oracle 초과** (0.488 > 0.483) → 학습 정책이 목표 외 지표에도 robust 일반화
3. **3 seeds 재현성**: std 모두 0.006 이하. 학습 알고리즘이 완전 재현 가능.

### 3.3 선행 JKSCI 수치 재현 분석

| 지표 | 선행 보고 | 본 논문 측정 | 재현율 |
|---|---|---|---|
| R-DWA F1 | 0.86 | 0.137 (strict) / 0.450 (substring) | 16% (strict) / **52% (substring)** |
| R-DWA EM | 0.78 | 0.000 | 구조적 0 |
| R-DWA Faith | 0.89 | 0.835 | **94%** |

**해석.** Faithfulness 94% 재현은 LLM 생성 품질이 양 구현에서 일관됨을 의미한다. F1의 gap은 선행 평가기의 엄격성 기준 미공개에 기인한다. 본 논문의 F1_substring 0.450은 선행 0.86의 52% 에 그치는데, 선행은 아마도 더 느슨한 (예: 부분 문자 일치, 동의어 매칭, 또는 subset 기반) 평가였을 가능성이 있다. 본 논문은 두 지표 병기로 이 불확실성을 명시한다.

## 5절 PPO 학습 동역학

### 5.1 3-seed 수렴

각 seed의 early R (첫 100 ep) 와 late R (마지막 100 ep):

| Seed | early R | late R | Δ | 판정 |
|---|---|---|---|---|
| 42 | 0.1988 | 0.2145 | +0.0157 | ✅ learning |
| 123 | 0.1970 | 0.2153 | +0.0183 | ✅ learning |
| 999 | 0.1964 | 0.2146 | +0.0182 | ✅ learning |

**모두 +0.016~0.018** 의 일관된 reward 상승 — 학습 알고리즘의 재현 신뢰도 확보.

### 5.2 오염된 F1 신호와 학습 타당성

캐시 구축 시점 평가기가 구두점 버그를 포함하고 있었으나, 본 논문은 L-DWA의 학습 타당성이 유지됨을 다음 논리로 정당화한다:

1. 버그는 F1 값을 *축소* (0.072→0.137 수정 후) 하였으며, *부호를 뒤집지는 않았다*.
2. 즉 가중치 조합 A가 B보다 좋은 보상을 주는 **순위** 는 버그 수정 전후에서 보존된다.
3. PPO는 보상 *크기* 가 아닌 *상대 비교* 로 정책을 개선한다.
4. 실제로 3 seed 전부 동일 수렴점 (R ≈ 0.215) 도달이 이를 간접 증거한다.
5. 정책의 최종 성능은 수정 평가기로 측정 (F1_strict 0.167) 하여 정확성 확보.

즉 **학습은 buggy F1로 진행되었으나 수렴된 정책은 수정 F1로 평가** 하였다. 이 분리는 과학적 타당성을 위한 필수 투명성이다.

### 5.3 정책 선택 가중치 분포 (seed 42 기준)

L-DWA의 평균 선택 가중치 ≈ **(α=0.25, β=0.30, γ=0.45)**. R-DWA의 (0.25, 0.45, 0.30) 대비 **β → γ 재분배**. Oracle의 극단적 γ-편향 (0.11, 0.16, 0.73) 에 완전 도달하지는 않으나 **중간 지점** 에서 수렴한다. 이는 entropy 계수 0.01 이 탐색을 유지하고 Dirichlet 평균의 smoothing 효과로 일종의 regularized 해를 학습함을 반영한다.

### 7.3 결과 해석: 도메인 특화 학습의 당위성

세 벤치마크 모두에서 **대학 학습 L-DWA가 R-DWA 및 Vector-only 대비 30~35% 저하** 한다. 원인:

1. **State feature 언어 의존성.** RuleBasedIntent의 정규식이 한국어 전용 (e.g. `[가-힣]{2,4}\s*교수`) 이어서 영어 질의에서 density = (0, 0, 0) 을 반환, 18-dim state의 상당 부분이 무신호 영역으로 진입.
2. **학습 환경-추론 환경의 구조 불일치.** 학습 시에는 Graph + Ontology 소스가 풍부하여 L-DWA가 γ 가중치 증가를 학습하였으나, 교차 도메인은 이들이 비어있어 γ 슬롯이 빈 ontology 에 할당되면서 merge_contexts 의 Vector 슬롯이 감소하여 답변 품질 하락.

### 7.4 HotpotQA 도메인-특화 재학습 시도 (Ch.6 추가 실험)

위 관찰을 뒤집어, HotpotQA 자체로 L-DWA를 재학습하였다 (cache 20K 엔트리 \$6, 학습 16분).
- **학습 trajectory FLAT**: R_early = 0.203, R_late = 0.202, Δ = −0.001
- 최종 성능: F1_strict 0.101 — Vector-only 0.102 와 사실상 동일.

원인: HotpotQA 는 Graph/Ontology 가 비어있어 merge_contexts가 α 만 실효적으로 반영. 이산 가중치 격자에서 α ≥ 0.2 인 모든 조합은 top-3 contexts 를 모두 포함 → 보상 동일 plateau → PPO gradient 소멸.

### 7.5 본 절의 시사점

- **L-DWA의 가치는 Triple-Hybrid RAG 구조가 실제로 작동하는 도메인 (복수 지식 소스 존재) 에 국한** 된다.
- 교차 도메인 naive transfer 는 불가능하며, **도메인별 (i) 다국어 state feature 혹은 (ii) 도메인별 cache 구축 + 재학습** 중 하나가 필요하다.
- 이 관찰은 본 논문의 기여 범위를 정직하게 경계 짓는다: 합성 대학 벤치마크에서의 R-DWA → L-DWA 정량 개선은 확실히 확보되었다 (§3). 범용 일반화는 §8.2 future work로 명시 위탁한다.

## 8절 한계 및 향후 연구

### 8.1 본 논문의 한계
1. **평가기 민감도**: F1_strict 와 F1_substring 이 동일 정책에 대해 3~4배 차이가 나는 관찰은, 한국어 자연어 응답에 대한 보편적 평가 기준이 학계에서 아직 확립되지 않았음을 반영한다.
2. **단일 학습 도메인**: 합성 대학 벤치마크에서만 end-to-end 학습을 검증하였다. §7의 교차 도메인은 transfer feasibility 만 시험.
3. **EM = 0 구조적**: 콤마 리스트 gold 형식으로 EM 이 모든 정책에서 0이다. Reward 식의 0.3·EM 항은 학습 신호로 기능하지 못하였다. 구성식 재설계 (예: 0.7·F1 + 0.3·Faith) 후 재학습이 향후 개선 기회이다.
4. **BERT intent classifier 미학습**: `src/intent/bert_classifier.py` 의 klue/bert-base 기반 분류기는 구현만 완료되어 있으며 학습은 향후 과제. intent_logits=(0,0,0) 상태에서도 L-DWA는 Oracle 99.4% 도달함을 고려할 때, BERT 학습은 주변 이득일 것이나 conditional 유형에서의 추가 개선 여지가 있다.
5. **Cache 구축 시 평가기 버그**: §1.4에 기술된 구두점 버그로 인해 cache 구축 시 F1 신호가 축소되었으나, §5.2 에서 논의한대로 순위 보존 하에 정책 수렴 타당성은 유지된다.

### 8.2 향후 연구 방향
1. **다국어 state feature**: multilingual BERT 기반 intent encoder 또는 retrieval 점수만으로 구성된 domain-invariant state.
2. **도메인별 fine-tuning**: 작은 per-domain cache (예: 500 QA × 66 = 33K 엔트리, ~$3) 에서 몇 천 episode 추가 학습.
3. **Reward 재설계**: EM 가중치 0.3을 F1 (list-aware) 또는 Faithfulness 로 재분배.
4. **Joint training**: 복수 도메인의 cache 를 결합 학습하여 L-DWA 가 내재적 도메인 signature 를 학습하는지 검증.
5. **평가 기준 통일**: 선행 `hybrid-rag-comparsion` 저장소에 본 논문의 버그 수정 (구두점 제거) 을 적용하여 F1 정의를 양 프로젝트에서 통일.

# 제Ⅶ장 결론 (v5 재작성)

## 1절 연구 요약

본 박사 논문은 Triple-Hybrid RAG 프레임워크의 지식 소스 간 가중치 결정을 Markov Decision Process 로 공식화하고 Proximal Policy Optimization 으로 학습하는 **Learned Dynamic Weighting Algorithm (L-DWA)** 을 제안하였다. 선행 연구 (Shin & Moon, 2025, JKSCI) 의 규칙 기반 R-DWA 의 네 가지 구조적 한계 — (i) 도메인 편향된 기본 가중치 테이블, (ii) 단일 λ 하이퍼파라미터, (iii) 유형 내 미세 구분 부재, (iv) 도메인 이전 불가 — 를 학습형 접근으로 해소하고, 합성 대학 5,000 QA 벤치마크에서 정량적 개선을 실증하였다.

## 2절 본 논문의 기여

### 2.1 방법론적 기여

**기여 1 — MDP 공식화.** Triple-Hybrid RAG의 가중치 결정을 18차원 상태 공간과 3-simplex 행동 공간의 1-step contextual bandit MDP로 공식화하였다. 이는 하이브리드 RAG 연구 흐름에서의 **최초 정식화** 이다.

**기여 2 — PPO-based L-DWA.** 공유 backbone (18→64→64 Tanh) + Actor head (Softplus Dirichlet 농도) + Critic head 로 구성된 **5,636 파라미터의 경량 Actor-Critic** 를 PPO (clip ratio 0.2, 10,000 episodes) 로 학습하여 3 seeds 전부 Total Reward 0.215 ± 0.002 의 일관된 수렴을 확보하였다.

### 2.2 엔지니어링 기여

**기여 3 — 오프라인 보상 캐시.** 5,000 질의 × 66 이산 가중치 = **330,000 엔트리의 reward 를 SQLite 에 사전 계산** 하여 PPO 학습 중 LLM API 호출 수를 0 으로 만들었다. 3 seeds 학습의 비용을 $288 → $33 (**−85%**), 실 wall-clock 을 50h+ → 1h PPO (**−98%**) 로 축소하였다.

**기여 4 — 이중 F1 평가 체계.** 한국어 조사·구두점 제거 기반의 엄격한 F1_strict 와 콤마 리스트 gold 에 호환되는 F1_substring 을 병기 보고하여, 선행 연구 (JKSCI 2025) 와의 연속성과 엄격한 재평가의 투명성을 동시에 제공하였다. 본 논문은 두 지표의 차이 자체를 **한국어 QA 평가의 방법론적 관찰** 로 기록한다.

### 2.3 실증적 기여

**기여 5 — L-DWA 성능 정량화 (5,000 QA, 3 seeds):**
- F1_strict **0.167 ± 0.001** (R-DWA 0.137 대비 **+21.9%**)
- F1_substring **0.488 ± 0.006** (R-DWA 0.450 대비 **+8.4%**)
- Faithfulness **0.866 ± 0.004** (+3.6%)
- Total R **0.215 ± 0.002** (+7.6%)

**기여 6 — Oracle 근접 성능.** L-DWA 는 per-query argmax Oracle 의 **F1_strict 99.4%, Faithfulness 97.5%** 에 도달하며, F1_substring 에서는 Oracle (0.483) 을 0.488 로 초과한다. 이는 학습 정책이 학습 목표 (Total R) 외 지표에도 robust 하게 일반화됨을 실증한다.

**기여 7 — Conditional 질의의 질적 개선.** Conditional 유형 (n=1,250) 에서 L-DWA 는 F1_substring 0.208 을 기록하여 R-DWA 의 0.090 대비 **+131% 상대 개선** 을 달성하며, Oracle 의 0.195 를 상회한다. 이는 per-query 학습형 가중치의 규칙 기반 대비 **질적 우월성** 에 대한 본 논문의 가장 강력한 증거이다.

**기여 8 — 교차 도메인 한계의 정직한 보고.** 합성 대학 학습 L-DWA 의 HotpotQA/MuSiQue/PubMedQA naive transfer 가 Vector-only 베이스라인 대비 −30 ~ −35% 저하함을 실측하여, **도메인 특화 학습이 본 프레임워크 가치의 전제** 임을 확립하였다. 범용 일반화는 향후 연구 과제로 명시 위탁한다.

## 3절 학술적·실용적 의의

### 3.1 학술적 의의

**하이브리드 RAG 연구에 강화학습 접근을 도입한 선도 사례.** 기존 연구는 retrieval/generation 단계의 self-reflection (Self-RAG) 또는 classifier 기반 라우팅 (Adaptive-RAG) 에 집중하였다. 본 논문은 **복수 지식 소스 간 연속 가중치 결정** 에 PPO 를 최초 적용하여 RAG-RL 결합의 새로운 방향을 제시한다.

**한국어 QA 평가의 실증적 기반.** F1_strict 와 F1_substring 의 0.137 vs 0.450 격차는 평가 기준에 따라 3배 이상 변동 가능함을 실증한다. 이는 한국어 NLP 평가 방법론 논의의 실증 데이터로 기여한다.

### 3.2 실용적 의의

**비용-시간 진입 장벽 완화.** 오프라인 보상 캐시로 RL 기반 RAG 튜닝의 비용을 학계·중소기업 감당 가능 수준 (일회 $33) 으로 낮추었다. 본 논문의 공개 GitHub 저장소 (https://github.com/sdw1621/triple-rag-phd) 는 `cache/university.sqlite` (16.8 MB) 와 3 seed 체크포인트를 포함하여 즉시 재현 / 재학습이 가능하다.

**도메인 특화 학습 레시피의 검증.** 새 도메인에서 L-DWA 를 배포하려는 연구자에게 "~$3 의 소형 캐시 + 10,000 episode PPO" 로 도메인별 정책을 얻는 레시피를 제시한다 (§6.7.4 참조). 이는 Graph/Ontology 가 풍부한 기업 내부 지식베이스 (법무, 의료, 금융) 에 적용 가능한 실무 경로이다.

## 4절 한계 및 향후 연구 방향

### 4.1 명시한 한계

1. **단일 도메인 end-to-end 학습.** 본 논문의 L-DWA 성능은 합성 대학 벤치마크에서만 end-to-end 검증되었다.
2. **도메인 이전의 naive transfer 실패.** §Ⅵ.7 의 정직한 보고.
3. **EM = 0 구조적.** Reward 식의 0.3·EM 항이 학습 신호로 기능하지 못함.
4. **BERT intent classifier 미학습.** 구현만 제시하고 intent_logits = (0, 0, 0) 으로 둠.
5. **Cache 구축 시 F1 평가기 버그.** §Ⅵ.1.4 및 §Ⅵ.5.2 에서 투명 기재.

### 4.2 향후 연구

1. **다국어 state feature.** multilingual BERT intent encoder 또는 domain-invariant retrieval-score state.

5. **선행 저장소와의 평가기 통일.** `hybrid-rag-comparsion` 에 본 논문의 버그 수정을 적용하여 F1 정의를 양 프로젝트에서 통일.
6. **BERT multi-label intent 학습 후 재평가.** 특히 Conditional 유형의 추가 개선 여지.
7. **더 큰 Actor-Critic.** 현 5,636 파라미터 네트워크의 표현력 한계와 스케일링 효과 조사.

## 5절 연구 과정의 자기 성찰

본 논문의 집필 과정에서 다음 두 가지의 실측-기반 관찰이 주목할 만하였다.

**첫째, F1 평가기 버그.** 쉼표·마침표 정규화 누락으로 F1_strict 가 ~50% 과소 측정되어 있던 버그를 논문 제출 직전 (§Ⅵ.1.4) 발견하였다. 수정 후 L-DWA 의 Oracle 도달률이 52% → 99.4% 로 급격히 상승하여 본 연구의 방법론적 주장이 훨씬 강화되었다. 평가기에 대한 지속적 비판적 검토의 중요성을 실증하는 사례이다.

**둘째, 교차 도메인 transfer 실패의 재프레이밍.** 초기 계획은 "L-DWA 가 4개 벤치마크 전부에서 SOTA 를 능가" 하는 서사였으나, 실측 결과 이는 실패하였다. 본 논문은 이 실패를 **숨기지 않고** "도메인 특화 학습이 기여의 본질" 로 재프레이밍 하였다. 정직한 부정적 결과가 방법론적 주장을 오히려 강화하는 박사 학위 연구의 전형적 사례라고 판단한다.

## 6절 맺음말

본 박사 논문은 Triple-Hybrid RAG 의 가중치 결정을 학습 가능한 정책으로 전환하여, 선행 연구의 규칙 기반 접근의 구조적 한계를 극복하고 합성 대학 벤치마크에서 재현 가능한 정량 개선을 실증하였다. 오프라인 보상 캐시는 강화학습 기반 RAG 연구의 실용적 진입 장벽을 완화하며, 이중 F1 평가 체계는 한국어 QA 연구의 방법론적 투명성에 기여한다. 교차 도메인 실험은 정직한 경계 조건을 제공하며 향후 연구의 명시적 출발점이 된다.

본 연구가 제공하는 코드, 캐시, 체크포인트의 완전 공개는 후속 연구자가 본 논문의 수치를 그대로 재현하거나, 자신의 도메인에 L-DWA 를 즉시 적용하거나, 본 논문의 한계를 개선하는 연구를 시작하는 모두에 기반이 되기를 기대한다.

## 1. 프레임워크 이름

| 용어 | 정의 | 영문 |
|---|---|---|
| **Triple-Hybrid RAG** | Vector, Graph, Ontology 3가지 지식 소스를 가중치 융합하는 RAG 아키텍처 (본 연구 및 선행 연구 공통) | Triple-Hybrid Retrieval-Augmented Generation |
| **R-DWA** | Rule-based Dynamic Weighting Algorithm. 선행 논문(JKSCI 2025)의 규칙 기반 가중치 결정 방식. 2단계 (질의 유형 → 기본 가중치, 밀도 신호 → 연속 조정) | Rule-based Dynamic Weighting Algorithm |
| **L-DWA** | **Learned** Dynamic Weighting Algorithm. 본 박사논문의 새 기여. PPO로 학습된 Actor-Critic 정책이 상태(18-dim)를 보고 가중치(α, β, γ)를 출력 | Learned Dynamic Weighting Algorithm |
| **Oracle** | **상한선 정책**. 각 질의에 대해 cache 안의 66개 가중치 중 total_reward 최고값을 선택. 실제 배포 불가능 (정답을 알아야 함). L-DWA의 개선 여지를 정량화하는 기준 | Oracle policy (upper bound) |

## 2. 지식 소스 (Source)

| 용어 | 정의 | 가중치 기호 |
|---|---|---|
| **Vector 소스** | 문서 임베딩 기반 FAISS 검색. 의미 유사도 상위 k개 문서 반환 | **α** |
| **Graph 소스** | NetworkX로 구축된 지식 그래프 상 BFS (3-hop). 개체-관계 경로 반환 | **β** |
| **Ontology 소스** | Owlready2 기반 OWL 온톨로지 + 규칙 fallback. 클래스 계층 + 제약 조건 만족 엔티티 반환 | **γ** |
| **Weight simplex** | (α, β, γ) ∈ Δ³, 즉 α + β + γ = 1, α,β,γ ≥ 0 | — |

## 3. 평가 지표

| 용어 | 정의 | 목적 |
|---|---|---|
| **F1_strict** | 한국어 조사 제거 + 구두점 제거 + NFC 정규화 후 **토큰 집합** 기반 F1. 본 논문의 **엄격한** 주 지표. | 재현 가능 + 엄격 |
| **F1_substring** | Gold를 쉼표로 분할하여 얻은 목록 항목들이 prediction에 부분문자열로 나타나는지 검사. **선행 논문(JKSCI 2025)이 사용한 것으로 추정**되는 loose 평가와 호환. | Journal 비교 |
| **EM (Exact Match)** | 정규화된 prediction과 gold의 완전 일치. 본 벤치마크에서 항상 0 (gold가 콤마 리스트 형식) → **본 논문에서 주 지표 아님** | 참고용 |
| **Faithfulness** | RAGAS 근사. Prediction의 각 문장이 retrieved context에 의해 지지되는 비율 | 답변 근거 |
| **Latency** | End-to-end 응답 시간 (초). 5초 초과 시 reward에 penalty | 실용성 |
| **Total Reward R** | `R = 0.5·F1_strict + 0.3·EM + 0.2·Faithfulness − 0.1·max(0, latency−5.0)`. PPO 학습의 목적 함수. | PPO 학습 목표 |

## 4. 질의 분류 체계

| 용어 | 정의 | 예시 | 비중 (gold_qa_5000) |
|---|---|---|---|
| **Simple** | 단일 속성 또는 관계 질의 | "김철수 교수의 소속 학과는?" | 2,000 (40%) |
| **Multi-hop** | 2개 이상의 관계를 연속 탐색해야 하는 질의 | "이영희 교수가 담당하는 과목을 수강하는 학과는?" | 1,750 (35%) |
| **Conditional** | 수치/범위/논리 제약을 포함하는 질의 (본 논문의 boundary query에 해당) | "40세 이하 컴공과 교수 중 AI 연구자는?" | 1,250 (25%) |
| ~~Boundary~~ | 구 용어 (v2). "Conditional"과 사실상 동일. **v5에서 Conditional로 통일**. |

## 5. PPO/RL 용어

| 용어 | 정의 |
|---|---|
| **MDP** | Markov Decision Process. 본 논문에서 Triple-Hybrid 가중치 결정을 1-step contextual bandit으로 공식화 |
| **State (18-dim)** | `[density(3) + intent_logits(3) + source_stats(9) + query_meta(3)]` (Ch.5 Eq.5-1) |
| **Action** | (α, β, γ) ∈ simplex. Dirichlet 분포에서 샘플링 |
| **Actor-Critic** | 공유 backbone (18→64→64 Tanh) + Actor head (Dirichlet 농도) + Critic head (가치). **5,636 파라미터** |
| **PPO** | Proximal Policy Optimization. Clip ratio 0.2, epochs 4, mini-batch 8 |
| **Rollout** | Episode당 32개 (질의, 가중치, 보상) 샘플 수집 |
| **Episode** | 1 rollout → policy update 사이클. 본 학습은 seed당 **10,000 episodes** |
| **Seed** | 난수 시드. 본 논문은 {42, 123, 999} 3개 seed로 mean ± std 보고 |

## 6. 엔지니어링 용어

| 용어 | 정의 |
|---|---|
| **Offline reward cache** | 5,000 질의 × 66 이산 가중치 = **330,000 엔트리**의 reward 사전 계산 SQLite DB. PPO 학습 중 API 호출 0. |
| **Discretization grid** | 가중치를 0.1 단위로 이산화. 0~10 정수 그리드, 합이 10인 삼각형 점 = 66개 |
| **Retrieval cache** | `evaluate_rerun.py`의 per-query retrieval 결과 pickle. 정책 여러 번 재측정 시 시간 절약 |
| **State cache** | `train_ppo.py`의 per-query state pickle. 여러 seed 학습 시 재사용 |
| **Snap-to-simplex** | 연속 Dirichlet 샘플을 정수 simplex (66 점 중 1)에 매핑 → cache 룩업 가능 |
